面向局域网聚会的简单文件共享

LANBucket 让您在同一网络中的设备之间轻松共享文件。 非常适合局域网聚会、大文件以及家里的快速传输。无需联网。

分布式传输。100% 无 Electron。

系统要求

  • Windows 10 版本 1809 或更高版本，或 Windows 11
  • x64 或 ARM64 处理器
  • 局域网连接（WiFi 或以太网）

为什么选择 LANBucket？

快速分布式传输

文件采用分布式技术传输，所有客户端共同分担负载，从而实现更快的整体传输速度。

非常适合局域网聚会

在多台设备间共享大文件，无需所有人同时从互联网下载。比传统文件共享快得多。

自动发现

网络中其他运行 LANBucket 的设备会被自动发现。无需手动配置。

仅限局域网

所有流量都保留在您的局域网内。无需互联网连接。您的文件保持完全私密和安全。

拖放操作

只需拖放文件或文件夹，即可立即与网络中的其他用户共享。

查看谁已连接

查看所有已连接的对等设备及其计算机名和用户名，准确了解您正在与谁共享。

工作原理

  1. 安装 LANBucket

    在连接到同一局域网（WiFi 或以太网）的 Windows 设备上下载并安装 LANBucket。

  2. 自动发现

    LANBucket 使用多播对等发现自动发现网络中的其他实例。

  3. 共享文件

    拖放文件或文件夹即可共享。其他用户将看到您共享的项目并可以下载。

  4. 下载

    浏览他人共享的文件并直接下载到您的设备。由于 LANBucket 采用分布式技术，即使多个用户同时下载，下载速度依然很快。

基于成熟技术构建

LANBucket 采用分布式文件共享技术，实现可靠而高效的传输。 该应用使用现代 Windows 技术构建，包括 WinUI 3 和 Windows App SDK， 带来原生、高性能的体验。

无需设置或配置。 LANBucket：是。DC++：否 — DC++ 要求您先添加要连接的中枢并设置共享列表，然后才能使用。D-LAN：部分支持 — D-LAN 上手很快，但您仍需先选择共享文件夹并确认网络设置。LocalSend：是。SMB 共享：否 — SMB 共享需要启用共享、设置权限并映射驱动器，通常还需要管理员权限。云存储：否 — 您必须创建账户、安装客户端并等待其同步，然后才能共享任何内容。Steam 本地共享：部分支持 — 您需要安装并登录 Steam，并启用相关的共享设置。USB 闪存盘：否 — 闪存盘必须为您的设备格式化，并逐一物理插入每台设备。

无需账户或登录。 LANBucket：是。DC++：是。D-LAN：是。LocalSend：是。SMB 共享：否 — SMB 需要有效的用户凭据或网络账户才能验证访问。云存储：否 — 云存储需要先创建并登录账户，然后才能传输文件。Steam 本地共享：否 — 您必须拥有 Steam 账户并登录才能使用任何 Steam 功能。USB 闪存盘：是。

分布式（多对多）传输。 LANBucket：是。DC++：部分支持 — DC++ 可以同时从多个对等端拉取文件，但只有在对方完成下载之后才行；LANBucket 允许从尚未拥有完整文件的对等端下载。D-LAN：是。LocalSend：否 — LocalSend 一次只在一台设备与另一台设备之间直接发送。SMB 共享：否 — SMB 共享由单一主机提供，因此所有人都从那一台机器下载。云存储：否 — 传输通过服务提供商的服务器进行，而不是在多个本地对等端之间直接进行。Steam 本地共享：否 — 内容来自单一主机，而非多个对等端共同分担负载。USB 闪存盘：否 — 闪存盘一次只能手动复制到一台机器。

常见问题

LANBucket 能通过互联网使用吗？

不能，LANBucket 仅在局域网中工作。所有设备必须连接到同一 WiFi 网络或以太网局域网。这是有意为之，目的是保持您的文件私密和安全。

它能在我的公司网络上使用吗？

许多公司网络出于安全原因会阻止客户端之间的通信，这可能导致 LANBucket 无法正常工作。 LANBucket 主要为家庭和私人网络设计。

如果多播被阻止，LANBucket 还能工作吗？

很可能可以。LANBucket 有一个使用中央服务器的后备对等发现机制，因此即使您的网络阻止了多播，设备之间仍可以相互发现并共享文件。 如果该机制无效，您可以通过手动输入要连接设备的 IP 地址来连接对等端。

我所有的客户端都在同一 WiFi 网络上，但有些无法相互看到。为什么？

许多 WiFi 路由器具有“AP 隔离”或“客户端隔离”功能，会阻止连接到同一 WiFi 网络的设备相互通信。 请检查您的路由器设置，看是否启用了此功能。 此外，我们还发现，有些无线路由器/接入点在客户端处于不同频段时不允许客户端之间的流量，即一个客户端 连接到 2.4GHz，另一个连接到 5GHz。如有可能，请尝试将所有客户端连接到 WiFi 网络的同一频段，或通过以太网连接以获得最佳效果。

我可以在使用 VPN 的情况下使用它吗？

VPN 连接可能会干扰局域网发现。您可能需要断开 VPN 才能使用 LANBucket。

如果我修改了正在共享的文件会怎样？

移动、删除或修改您正在共享的文件会导致其他人无法下载。 请在其他人完成下载之前保持共享文件不动。

我的防火墙会阻止 LANBucket 吗？

Windows Defender 防火墙可能会阻止 LANBucket 发现设备。请确保将网络配置文件设置为“专用”，并在首次启动出现提示时允许 LANBucket 通过防火墙。

有 Linux 或 MacOS 支持吗？

暂时没有，但我们正在考虑。目前 LANBucket 使用 Windows 专用的 UI 框架构建，因此支持 Linux/MacOS 需要大量重写。遗憾的是，LANBucket 也不兼容 Wine/Proton，因为我们使用了那里同样未实现的 Windows API。

什么是实时压缩？

LANBucket 会在文件发送时自动压缩、在文件到达时自动解压，全部实时完成。 您无需先打包成 zip。对于压缩效果好的文件（例如某些电子游戏）， 您可能会观察到显著更快的传输速度。如果文件压缩效果不佳，压缩会自动关闭。
